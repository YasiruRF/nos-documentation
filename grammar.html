Language Grammar Definition

NOS uses **ANTLR4** for lexing and parsing source `.nos` files. The grammar is defined in two parts: a Lexer definition (NOSLexer.g4) and a Parser definition (NOSParser.g4).

Lexer Modes & Context Shifting

Since NOS allows embedding raw Python code blocks, the lexer handles shifting parser contexts. When the lexer encounters the token combination -> {, it pushes a new lexer state (PYTHON_MODE) to parse the Python content cleanly. 

lexer grammar NOSLexer;

// When matching -> {, transition into Python mode
ARROW_BRACE : '->' [ \t\r\n]* '{' -> pushMode(PYTHON_MODE);

mode PYTHON_MODE;

// Match non-brace characters inside the code block
PYTHON_BLOCK : [^}]+ ;

// When matching }, return back to standard DSL mode
CLOSE_BRACE : '}' -> popMode;

AST Node Generation

At compile time, the parser walks the generated concrete syntax tree using listener classes and constructs a custom Abstract Syntax Tree (AST) defined in nos.ast.nodes. This AST is then verified by the semantic analyzer and translated into compliant ROS2 source packages.
