Managed Lifecycle Nodes

Goal: Create, compile, and transition a Managed Lifecycle Node using the NOS DSL.

Tutorial level: Intermediate

Time: 15 minutes

Background

Standard ROS2 Nodes start all execution routines in their constructors. However, for complex safety-critical robotic systems, it is essential to manage node activation states deterministically.

ROS2 Managed Lifecycle Nodes solve this by introducing a state machine (Unconfigured, Inactive, Active, Finalized). For design specifics, see the Official ROS2 Lifecycle Design Document.

Prerequisites

Ensure you have completed the basic talker/listener guide in the Quickstart Guide.

Tasks

1. Write the Managed Node

Create a file named camera.nos and declare lifecycle: managed inside your node block:

package camera_system
version "0.1.0"

node CameraDriver {
    lifecycle: managed

    publications {
        image: sensor_msgs::Image @topic("/camera/image")
    }

    on_configure -> {
        # Initialize resources (e.g. open camera hardware connection)
        self.get_logger().info("Hardware connection configured!")
    }

    on_activate -> {
        # Start publications or timer execution loops
        self.get_logger().info("Image streaming activated.")
    }

    on_deactivate -> {
        # Stop publications
        self.get_logger().info("Image streaming paused.")
    }

    on_cleanup -> {
        # Clean up resources
        self.get_logger().info("Hardware resources cleaned up.")
    }
}

Code Breakdown

  • lifecycle: managed: Instructs the compiler to inherit from rclpy.lifecycle.LifecycleNode rather than standard Node.
  • on_configure, on_activate, on_deactivate, on_cleanup: Corresponds to standard transition states. The generated wrapper handles routing transition return values (TransitionCallbackReturn.SUCCESS) automatically.

2. Compile, Build, and Run

Compile the node:

nos compile camera.nos --output ./src

Build the package using Colcon from your workspace root:

colcon build --packages-select camera_system
source install/setup.bash
ros2 run camera_system CameraDriver

3. Introspect and trigger state transitions

In a new terminal window, use the official ROS2 lifecycle CLI commands to verify state changes:

# Check the current state of the node (should return: unconfigured [1])
ros2 lifecycle get /camera_driver

# Configure the node (triggers on_configure)
ros2 lifecycle set /camera_driver configure

# Activate the node (triggers on_activate, begins publishing)
ros2 lifecycle set /camera_driver activate

# Pause the node (triggers on_deactivate)
ros2 lifecycle set /camera_driver deactivate
