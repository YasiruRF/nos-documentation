Managed Lifecycle Nodes

ROS2 Managed Lifecycle Nodes allow deterministic control over the state of a node. Rather than starting communications and node processes during construction, you can trigger transition states (e.g., config, activate, deactivate, shutdown) via lifecycle management.

(For design specifics, read the official ROS2 Managed Lifecycle Design Article).

Declaring a Managed Node in NOS

Simply declare lifecycle: managed inside your node block:

node CameraDriver {
    lifecycle: managed

    on_configure -> {
        self.get_logger().info("Hardware configured!")
    }

    on_activate -> {
        self.get_logger().info("Sensor active, starting publications.")
    }

    on_deactivate -> {
        self.get_logger().info("Sensor paused.")
    }

    on_cleanup -> {
        self.get_logger().info("Hardware cleaned up.")
    }
}

Under the Hood Mappings

NOS compiles the node into a subclass of rclpy.lifecycle.LifecycleNode. It automatically manages the standard transition hooks and handles transition return values (TransitionCallbackReturn.SUCCESS).

  • Communications Setup: Publications and subscriptions are configured in the on_configure state transition callback to ensure topics are only instantiated when the node is in the configured state.
  • User Callbacks: The lifecycle transition methods are mapped directly to your custom on_configure, on_activate, on_deactivate, and on_cleanup code blocks.
