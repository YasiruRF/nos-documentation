Launch System DSL

Goal: Orchestrate and run multiple nodes, parameters, and namespaces using the declarative Launch DSL.

Tutorial level: Intermediate

Time: 15 minutes

Background

In ROS2, running robot stacks requires launching multiple executable nodes concurrently. Rather than starting each node in a separate terminal, ROS2 uses launch files to define dependencies, namespaces, and node composition.

(For design specifics, read the official ROS2 Launch Concept documentation).

Prerequisites

Ensure you have created your first package using the Quickstart Guide.

Tasks

1. Write the launch configuration

Define your multi-node launcher inside the launch statement block of your `.nos` file:

package navigation_system
version "0.1.0"

launch RobotBringup {
    arguments {
        use_sim_time: bool = true
    }

    # Group components under a shared namespace
    group sensors @namespace("sensors") {
        lidar: LidarDriver {
            parameters: { use_sim: ${use_sim_time} }
        }
        camera: CameraDriver {
            parameters: { fps: 30 }
        }
    }

    # Setup composable component container process
    container comp_container @name("nav_container") {
        filter: PointCloudFilter
        planner: PathPlanner
    }
}

Code Breakdown

  • arguments { ... }: Declares a launch parameter. Compiles to a DeclareLaunchArgument instance.
  • group sensors @namespace("sensors"): Enforces a `/sensors` namespace prefix on both LidarDriver and CameraDriver nodes. Compiles to a GroupAction.
  • container comp_container @name(...): Spawns a ComposableNodeContainer, running both PointCloudFilter and PathPlanner as pluggable components in the same process to eliminate IPC overhead.

2. Compile and Build

Compile the launch system:

nos compile navigation.nos --output ./src

The compiler generates a standard RobotBringup.launch.py file inside the package's launch/ directory.

Build and source the workspace:

colcon build --packages-select navigation_system
source install/setup.bash

3. Run the Launch configuration

Execute the generated launch file using the standard ROS2 launch utility:

ros2 launch navigation_system RobotBringup.launch.py use_sim_time:=true
