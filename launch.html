Launch System DSL

ROS2 launch files allow starting, configuration, and coordination of multiple nodes. (For standard designs, see the official ROS2 Launch Concept documentation).

NOS features a dedicated launch statement block, mapping declarative node deployments, groups, arguments, and composable component containers directly to Python ROS2 launch descriptions.

Launch Arguments

Declare runtime configurable arguments inside the arguments block:

launch VehicleBringup {
    arguments {
        use_sim_time: bool = true
        max_speed: float = 1.0
    }
}

These compile to standard DeclareLaunchArgument nodes in the launch description.

Group Nodes and Namespaces

Group executable nodes under namespaces or scopes using group blocks:

group navigation @namespace("nav_stack") {
    lidar: LidarProcessor {
        scan_topic: "/scan"
    }
    planner: PathPlanner {
        sim_time: ${use_sim_time}
    }
}

This maps to GroupAction containing the registered Node wrappers.

Composable Component Containers

In ROS2, multiple nodes can be loaded as plugins into a single OS process (container) to reduce IPC latency. (For more details, see the official ROS2 Composition Guide).

Declare component containers natively using container:

container sensors_container @name("sensor_manager") {
    filter: PointCloudFilter
    fusion: SensorFusion
}

This compiles to a ComposableNodeContainer running the requested composable nodes inside a single process manager.
