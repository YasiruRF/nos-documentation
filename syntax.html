Nodes & Layout Syntax

Package Decl

Every NOS file must start with a package statement. Packages correspond to ROS2 package names and encapsulate all declared nodes and launch layouts. 

package package_name
version "1.0.0"

Nodes

A node represents a standard executable class. NOS transpiles this into a subclass of rclpy.node.Node. 

node NodeName {
    # Blocks inside
}

Parameters & Validation

The parameters block maps to ROS2 Node Parameters. You declare parameter names, types, default values, and optional range validation annotations. 

parameters {
    # Default types
    name: string = "robot_one"
    max_linear_speed: float = 0.5 @range(0.0, 2.0)
    update_rate: int = 10 @range(1, 100)
    mode: int = 1 @one_of(1, 2, 3)
}
🔗 ROS2 Parameters Mapping

NOS compiles these into official ROS2 ParameterDescriptor objects. The ranges are registered internally using FloatingPointRange or IntegerRange, which automatically enables ROS2 dynamic parameter safety validation.

Publications & Subscriptions

Topics are defined inside subscriptions and publications blocks, matching specific ROS2 message interfaces. 

subscriptions {
    scan: sensor_msgs::LaserScan @topic("/scan")
}

publications {
    cmd_vel: geometry_msgs::Twist @topic("/cmd_vel") @qos(reliable, depth=10)
}

QoS Configurations

The @qos annotation maps directly to ROS2 Quality of Service Profiles. Supported arguments include:

  • Reliability: reliable, best_effort (Corresponds to ReliabilityPolicy).
  • Durability: transient_local, volatile (Corresponds to DurabilityPolicy).
  • History: keep_last, keep_all (Corresponds to HistoryPolicy).
  • Depth: depth=integer_value (Corresponds to Queue size).
