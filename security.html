Security Code Sanitizer

Because NOS compiles and injects user-written Python code blocks into standard classes, the compiler includes an integrated AST code sanitizer to prevent security exploits.

AST-Based Verification

Unlike primitive compilers that check for dangerous terms using simple string searches (which produce false positives when keywords appear inside logging statements or comments), NOS compiles the python block into a Python Abstract Syntax Tree (AST) using the standard ast module.

The compiler walks the generated syntax tree, inspecting the nodes to verify code safety.

Blocked Operations & API Restrictions

The sanitizer maintains strict validation checks:

  • Dangerous Python Builtins: Code containing eval(), exec(), __import__, or bytecode manipulation triggers immediate validation failure.
  • Strict Mode Checks: When enabled, filesystems operations (e.g. .read(), .write()) and socket networking calls are validated.

Robust Attribute Validation

By checking actual ast.Attribute nodes, functions like list removal (my_list.remove("item")) or log string outputs do not trigger security warnings, even though they contain keywords like "remove" or "read".
