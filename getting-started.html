Writing a simple node (NOS)

Goal: Create, compile, and run your first declaratively configured ROS2 node in NOS.

Tutorial level: Beginner

Time: 10 minutes

Background

In this tutorial, you will create a simple data filter node. In standard ROS2, writing a node in Python requires manually importing libraries, inheriting from Node, declaring parameters in the constructor, setting up subscriptions and publications, and manually building configuration files.

NOS lets you declare this structure simply and generates the full buildable ROS2 package automatically.

Prerequisites

Before starting, ensure you have installed the nos package according to the Installation Guide, and that your ROS2 environment is sourced.

Tasks

1. Write the declarative source file

Create a file named filter_system.nos and add the following code:

package filter_system
version "0.1.0"

node ValueFilter {
    parameters {
        threshold: int = 10
    }

    subscriptions {
        input_data: std_msgs::Int32 @topic("/numbers/raw")
    }

    publications {
        output_data: std_msgs::Int32 @topic("/numbers/filtered")
    }

    on_init -> {
        self.get_logger().info("ValueFilter Node initialized!")
    }

    on on_input_data_received(msg: std_msgs::Int32) -> {
        if msg.data > self.threshold:
            out = Int32()
            out.data = msg.data
            self.output_data.publish(out)
    }
}

Code Breakdown

  • package filter_system: Declares the target package container.
  • node ValueFilter: Declares a new ROS2 node named ValueFilter.
  • parameters { threshold: int = 10 }: Automatically declares a node parameter in the ROS2 parameter server with a default value of 10.
  • subscriptions { ... }: Declares a subscriber listening to `std_msgs/Int32` on the `/numbers/raw` topic.
  • publications { ... }: Declares a publisher sending `std_msgs/Int32` on the `/numbers/filtered` topic.
  • on_init -> { ... }: Run at start.
  • on on_input_data_received(msg) -> { ... }: Automatically bound as the subscriber callback.

2. Compile the package with the CLI

Run the compiler in your terminal, specifying an output directory:

nos compile filter_system.nos --output ./src

The compiler generates a full ROS2 package layout under ./src/filter_system.

3. Build the package with Colcon

Navigate to your workspace root and compile the package using the ROS2 build tool Colcon:

colcon build --packages-select filter_system
source install/setup.bash

4. Run the node

Run the generated node executable using the standard ROS2 runner command:

ros2 run filter_system ValueFilter
