Quickstart Guide

Let's create a minimal system containing a single Node that subscribes to raw integer data, filters out values, and republishes the result.

Step 1: Write the `.nos` file

Create a file named filter_system.nos:

package filter_system
version "0.1.0"

node ValueFilter {
    parameters {
        threshold: int = 10
    }

    subscriptions {
        input_data: std_msgs::Int32 @topic("/numbers/raw")
    }

    publications {
        output_data: std_msgs::Int32 @topic("/numbers/filtered")
    }

    on_init -> {
        self.get_logger().info("Filter Node active!")
    }

    on on_input_data_received(msg: std_msgs::Int32) -> {
        # Check against value parameters
        if msg.data > self.threshold:
            # Create a new Int32 message and republish
            out = Int32()
            out.data = msg.data
            self.output_data.publish(out)
            self.get_logger().info(f"Published filtered value: {msg.data}")
    }
}

Step 2: Compile the package

Run the nos compile CLI command, pointing to the source file and specifying the output package root:

nos compile filter_system.nos --output ./src

Step 3: Analyze the output structure

NOS automatically creates a standard ROS2 `ament_python` package inside your folder:

src/
└── filter_system/
    ├── filter_system/
    │   ├── __init__.py
    │   └── ValueFilter.py      # Generated ROS2 python node
    ├── launch/                 # Output launch configs (if declared)
    ├── resource/
    │   └── filter_system       # Package marker file
    ├── package.xml             # Declared executable dependencies
    ├── setup.cfg
    └── setup.py                # Console script entry points

Step 4: Build and Source

Use your ROS2 workspace build tool (Colcon) to build the generated package:

# Run from your colcon workspace root (e.g. ~/ros2_ws)
colcon build --packages-select filter_system
source install/setup.bash

Step 5: Run the Node

Run the compiled Node executable directly from ROS2:

ros2 run filter_system ValueFilter
