Callbacks & Code Blocks

NOS combines declarative architecture configuration with raw Python code injection. Any block starting with -> { switches the parser context into Python mode, capturing the contents and compiling it directly into Python methods inside the node class.

Builtin Lifecycle Events

You can define hooks for standard events:

  • on_init: Executed when the node finishes class construction (inside the node's __init__ constructor).
  • on_shutdown: Executed when the node receives a shutdown signal (inside rclpy.shutdown() hooks).
on_init -> {
    self.get_logger().info("System has started!")
    self.counter = 0
}

on_shutdown -> {
    self.get_logger().info("System closing down.")
}

Communication Callback Naming Mappings

When subscription topics receive incoming data, the node wrapper executes callbacks. You can name your handler in two ways:

  1. Using the prefix on_<subscription_name>_received: 
    subscriptions {
    raw_data: std_msgs::Int32 @topic("/raw")
}
on on_raw_data_received(msg: std_msgs::Int32) -> {
    # Callback logic
}
  2. Using the simple name <subscription_name>_received: 
    on raw_data_received(msg: std_msgs::Int32) -> {
    # Callback logic
}

NOS automatically resolves both formats and maps them securely to the topic wrapper.

⚠️ Referencing Class Scope (Self)

Because code blocks are transpiled directly into Python class methods, you must use standard Python class scope structures. References to publishers, parameters, and other node utilities must be prefixed with self..
